Multi-Application Support System
MASS - Multiple Application Support System

The main idea of this application is to build a whole application in a modular way - all the application functionality is provided by composing different subsets of plugins, where each plugin should have responsibility over a single part of the functionality.

How it works

Main application which are storing in folder "/Application" loads all plugins from folder "/Application/Plugins".

Among all loaded plugins it searches the one, which inherits interface ICorePlugin. If it finds one - it will send all other loaded plugins to it by using AddPlugin(). After all main application calls Run() method.

If plugin implements ICorePlugin interface it should take care of whole system plugins interactions. Here's it's code:

class ICorePlugin
{
public:
virtual ~ICorePlugin() {}
virtual void AddPlugin(QObject* instance, QJsonObject* meta) = 0;
virtual void Run(QWidget* parentWidget) = 0;
};

So, that's all that designed in a non-modular way - everything else is contained in plugins :)

Plugins implementation aspect

All plugins are loaded with using of QPluginLoader and should contain plugin specific macroses. If you use QtCreator then you can use the custom wizard (it is here). It generates all plugin-specific code for you and you don't even need to figure out how all this stuff works.

Development

Currently there are exists only one core plugin (implementation of ICorePlugin) - MainMenuModel. There are implemented:

  • Specific plugins architecture
  • Plugins linker
  • Widgets management

All plugins (including core plugin) contains in "/Plugins". Description of the plugins-related theme is separated from the current page because theoretically, you could make your own implementation of core plugin.
